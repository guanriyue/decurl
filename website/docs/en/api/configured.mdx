# @guanriyue/decurl/configured

`@guanriyue/decurl/configured` creates hooks and runtime configuration components bound to the same search store.

The default hooks can be used directly. The configured entry is better suited for these scenarios:

- The application has multiple React Router runtimes and needs isolated search state.
- You want to hold the `store` explicitly.
- You want components to avoid direct dependency on React Router `useLocation`.
- You need to configure `flushDelay` or `flushMode`.

```tsx
import { createReactRouterSearch } from '@guanriyue/decurl/configured';
```

## createReactRouterSearch

```ts
createReactRouterSearch(options?): ReactRouterSearch
```

`createReactRouterSearch(options?)` creates hooks and configuration components bound to the same store.

```tsx
const search = createReactRouterSearch();
```

### Return Object

| API | Description |
| --- | --- |
| `store` | The search store created by the current factory. |
| `Provider` | Provides the bound store and wires the React Router runtime. |
| `ContextProvider` | Compatibility alias for `Provider`. |
| `RuntimeConfigurer` | Wires the current React Router runtime in component routers. |
| `RouterRuntimeConfigurer` | Wires the runtime with a data router instance. |
| `useSearchValues` | Reads and updates multiple search values. |
| `useSearchValue` | Reads and updates a single search value. |

Configured hooks must be used with the runtime/store provided by the same `search` instance.

## Runtime Configuration

Configured hooks do not configure the runtime automatically. Before using them, render `Provider`, `RuntimeConfigurer`, or `RouterRuntimeConfigurer` from the same `search` instance.

### Provider

`Provider` is the most common wiring method. When using component routers such as `BrowserRouter`, place `Provider` inside the Router and wrap components that consume configured hooks.

```tsx
import { BrowserRouter } from 'react-router';
import { createReactRouterSearch } from '@guanriyue/decurl/configured';

const search = createReactRouterSearch();

export const App = () => {
  return (
    <BrowserRouter>
      <search.Provider>
        <SearchPanel />
      </search.Provider>
    </BrowserRouter>
  );
};
```

If you use a data router instance, pass `router` to `Provider`:

```tsx
<search.Provider router={router}>
  <RouterProvider router={router} />
</search.Provider>
```

### RuntimeConfigurer

`RuntimeConfigurer` only wires the runtime and does not wrap children. It is useful when you do not use `Provider`, but still want to wire the runtime explicitly.

```tsx
<BrowserRouter>
  <search.RuntimeConfigurer />
  <SearchPanel />
</BrowserRouter>
```

### RouterRuntimeConfigurer

`RouterRuntimeConfigurer` accepts a data router instance and connects it to the current search store.

```tsx
<search.RouterRuntimeConfigurer router={router} />
<RouterProvider router={router} />
```

## Hooks

Configured hooks are called the same way as the default hooks, but they are bound to the store returned by the current `createReactRouterSearch` object.

```tsx
const SearchPanel = () => {
  const [values, setValues] = search.useSearchValues(searchFields);

  return (
    <button
      type="button"
      onClick={() => {
        setValues({ page: values.page + 1 });
      }}
    >
      Next page
    </button>
  );
};
```

Single-field reading:

```tsx
const PageButton = () => {
  const [page, setPage] = search.useSearchValue(searchFields.page);

  return (
    <button
      type="button"
      onClick={() => {
        setPage(page + 1);
      }}
    >
      Next page
    </button>
  );
};
```

Setter semantics such as patch values, `setValues(undefined)` clearing the current Search Fields, `null` / `undefined` deleting fields, and navigate options are the same as the default [@guanriyue/decurl](./react-router) entry.

### Composing Pagination

Pagination can be bound to the current search store with `createUseSearchPagination`:

```tsx
import { createReactRouterSearch } from '@guanriyue/decurl/configured';
import { createUseSearchPagination } from '@guanriyue/decurl/pagination';

const search = createReactRouterSearch();

const useSearchPagination = createUseSearchPagination({
  useSearchValues: search.useSearchValues,
});
```

The object returned by `createReactRouterSearch` does not include a pagination hook by itself. See [@guanriyue/decurl/pagination](./pagination#createusesearchpagination) for the complete API.

## Options

`createReactRouterSearch` accepts store options.

| Option | Type | Default | Description |
| --- | --- | --- | --- |
| `initialLocation` | `{ pathname: string; search: string }` | Current runtime location | Initial location. Usually you do not need to pass it manually. |
| `flushDelay` | `number` | `100` | Delay in milliseconds before search is written to the URL. Negative values are treated as `0`. |
| `flushMode` | `'throttle' \| 'debounce'` | `'throttle'` | Flush scheduling mode. |

```ts
const search = createReactRouterSearch({
  flushDelay: 80,
  flushMode: 'debounce',
});
```

### flushMode

`flushMode: 'throttle'` starts a fixed window after the first update. Repeated updates inside the window do not delay the flush.

`flushMode: 'debounce'` restarts the timer on every update. Continuous updates delay the flush.

Business code usually does not need a very long `flushDelay`. If you only want to observe render or URL flush behavior, temporarily increase it in tests or demos.

## Render Behavior

Configured hooks do not directly depend on React Router `useLocation`. In some route update scenarios, this can reduce extra renders. Whether extra renders are completely avoided still depends on the component tree and React optimizations such as `React.memo`.

For a fuller observation example, see [Bound Store](/guide/react-router/configured-store).
