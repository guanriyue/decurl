# @guanriyue/decurl/configured

`@guanriyue/decurl/configured` 用于创建一组绑定同一个 search store 的 hooks 与 runtime 配置组件。

默认 hooks 可以直接使用；`@guanriyue/decurl/configured` 更适合这些场景：

- 应用中存在多套 React Router runtime，需要隔离 search state。
- 希望显式持有 `store`。
- 希望组件不直接依赖 React Router `useLocation`。
- 需要配置 `flushDelay` 或 `flushMode`。

```tsx
import { createReactRouterSearch } from '@guanriyue/decurl/configured';
```

## createReactRouterSearch

```ts
createReactRouterSearch(options?): ReactRouterSearch
```

`createReactRouterSearch(options?)` 创建一组绑定同一个 store 的 hooks 与配置组件。

```tsx
const search = createReactRouterSearch();
```

### 返回对象

| API | 说明 |
| --- | --- |
| `store` | 当前 factory 创建的 search store。 |
| `Provider` | 提供绑定 store，并接线 React Router runtime。 |
| `ContextProvider` | `Provider` 的兼容别名。 |
| `RuntimeConfigurer` | 在组件式 Router 内接线当前 React Router runtime。 |
| `RouterRuntimeConfigurer` | 使用 data router instance 接线 runtime。 |
| `useSearchValues` | 读取和更新多个 search values。 |
| `useSearchValue` | 读取和更新单个 search value。 |

configured hooks 必须和同一个 `search` 实例提供的 runtime/store 配套使用。

## Runtime 配置

configured hooks 不会自动配置 runtime。使用它们之前，需要先渲染同一个 `search` 实例上的 `Provider`、`RuntimeConfigurer` 或 `RouterRuntimeConfigurer`。

### Provider

`Provider` 是最常用的接线方式。使用 `BrowserRouter` 等组件式 Router 时，`Provider` 应放在 Router 内部，并包裹消费 configured hooks 的组件。

```tsx
import { BrowserRouter } from 'react-router';
import { createReactRouterSearch } from '@guanriyue/decurl/configured';

const search = createReactRouterSearch();

export const App = () => {
  return (
    <BrowserRouter>
      <search.Provider>
        <SearchPanel />
      </search.Provider>
    </BrowserRouter>
  );
};
```

如果使用 data router instance，可以把 `router` 传给 `Provider`：

```tsx
<search.Provider router={router}>
  <RouterProvider router={router} />
</search.Provider>
```

### RuntimeConfigurer

`RuntimeConfigurer` 只负责接线 runtime，不额外包裹 children。它适合不使用 `Provider`、但仍希望显式完成 runtime 接线的场景。

```tsx
<BrowserRouter>
  <search.RuntimeConfigurer />
  <SearchPanel />
</BrowserRouter>
```

### RouterRuntimeConfigurer

`RouterRuntimeConfigurer` 接收 data router instance，并把它接到当前 search store。

```tsx
<search.RouterRuntimeConfigurer router={router} />
<RouterProvider router={router} />
```

## Hooks

configured hooks 和默认 hooks 的调用方式一致，但它们绑定到当前 `createReactRouterSearch` 返回对象的 store。

```tsx
const SearchPanel = () => {
  const [values, setValues] = search.useSearchValues(searchFields);

  return (
    <button
      type="button"
      onClick={() => {
        setValues({ page: values.page + 1 });
      }}
    >
      Next page
    </button>
  );
};
```

单字段读取：

```tsx
const PageButton = () => {
  const [page, setPage] = search.useSearchValue(searchFields.page);

  return (
    <button
      type="button"
      onClick={() => {
        setPage(page + 1);
      }}
    >
      Next page
    </button>
  );
};
```

setter 的 patch、`setValues(undefined)` 清除当前 Search Fields、`null` / `undefined` 删除字段、navigate options 等语义与 [@guanriyue/decurl](./react-router) 默认 hooks 一致。

### 组合 Pagination

Pagination 可以通过 `createUseSearchPagination` 绑定到当前 search store：

```tsx
import { createReactRouterSearch } from '@guanriyue/decurl/configured';
import { createUseSearchPagination } from '@guanriyue/decurl/pagination';

const search = createReactRouterSearch();

const useSearchPagination = createUseSearchPagination({
  useSearchValues: search.useSearchValues,
});
```

`createReactRouterSearch` 的返回对象本身不包含 pagination hook。完整 API 见 [@guanriyue/decurl/pagination](./pagination#createusesearchpagination)。

## Options

`createReactRouterSearch` 可接收 store options。

| 选项 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `initialLocation` | `{ pathname: string; search: string }` | 当前 runtime location | 初始 location。通常不需要手动传入。 |
| `flushDelay` | `number` | `100` | search 写入 URL 前的延迟毫秒数。负数会按 `0` 处理。 |
| `flushMode` | `'throttle' \| 'debounce'` | `'throttle'` | flush 调度模式。 |

```ts
const search = createReactRouterSearch({
  flushDelay: 80,
  flushMode: 'debounce',
});
```

### flushMode

`flushMode: 'throttle'` 表示第一次更新后启动固定窗口，窗口内重复更新不会推迟 flush。

`flushMode: 'debounce'` 表示每次更新都会重新计时，持续更新会推迟 flush。

业务中通常不需要设置很长的 `flushDelay`。如果只是为了观察渲染或 URL flush 行为，可以在测试或 demo 中临时放大。

## Render behavior

configured hooks 不直接依赖 React Router `useLocation`。在一些路由更新场景下，这可以减少额外渲染；是否完全避免额外渲染，还取决于组件树结构以及是否使用 `React.memo` 等 React 优化手段。

更完整的观测示例见 [绑定 Store](/zh/guide/react-router/configured-store)。

## Types

| 类型 | 说明 |
| --- | --- |
| `CreateReactRouterSearchOptions` | `createReactRouterSearch` 的 options 类型。 |
| `ReactRouterSearch` | `createReactRouterSearch` 的返回类型。 |
| `SetSearchValues<TFields>` | `useSearchValues` setter 类型。 |
| `SetSearchValue<TCodec>` | `useSearchValue` setter 类型。 |
| `UseSearchValuesResult<TFields>` | `useSearchValues` 返回元组类型。 |
| `UseSearchValueResult<TCodec>` | `useSearchValue` 返回元组类型。 |
