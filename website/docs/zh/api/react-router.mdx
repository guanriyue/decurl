# @decurl/react-router

`@decurl/react-router` 主入口提供开箱即用的 React Router hooks。它们会在 hook 内部接线当前 React Router runtime，不需要手动放置 Provider。

```tsx
import { useSearchValue, useSearchValues } from '@decurl/react-router';
```

如果需要绑定 store、多 React Router runtime 隔离，或者减少组件对 React Router `useLocation` 的直接依赖，见 [@decurl/react-router/configured](./configured)。

## useSearchValues

`useSearchValues(searchFields)` 读取和更新多个 search values。

```tsx
import { useSearchValues } from '@decurl/react-router';

const SearchPanel = () => {
  const [values, setValues] = useSearchValues(searchFields);

  return (
    <button
      type="button"
      onClick={() => {
        setValues({ page: values.page + 1 });
      }}
    >
      Next page
    </button>
  );
};
```

`setValues(patch, options?)` 接收 patch。只会更新传入字段，未传入字段保持不变。

patch 也可以是 updater：

```tsx
setValues((previousValues) => ({
  page: previousValues.page + 1,
}));
```

## useSearchValue

`useSearchValue(codec)` 读取和更新单个 field。传入的 codec 必须已经拥有确定的 `name`，通常来自 `defineFields` 的返回值。

```tsx
import { useSearchValue } from '@decurl/react-router';

const PageButton = () => {
  const [page, setPage] = useSearchValue(searchFields.page);

  return (
    <button
      type="button"
      onClick={() => {
        setPage(page + 1);
      }}
    >
      Next page
    </button>
  );
};
```

`setValue(patch, options?)` 同样支持直接值或 updater：

```tsx
setPage((previousPage) => previousPage + 1);
```

## Navigate options

setter 的第二个参数会透传给 React Router navigate。

| 选项 | 类型 | 说明 |
| --- | --- | --- |
| `replace` | `boolean` | 是否替换当前 history entry。 |
| `preventScrollReset` | `boolean` | 是否阻止滚动位置重置。 |

```tsx
setPage(1, { replace: true, preventScrollReset: true });
```

## Types

| 类型 | 说明 |
| --- | --- |
| `SetSearchValues<TFields>` | `useSearchValues` setter 类型。 |
| `SetSearchValue<TCodec>` | `useSearchValue` setter 类型。 |
| `UseSearchValuesResult<TFields>` | `useSearchValues` 返回元组类型。 |
| `UseSearchValueResult<TCodec>` | `useSearchValue` 返回元组类型。 |
