description: createURLSearchParamsCodec 将 Search Fields 编译为 URLSearchParams 的 decode 和 patch encode 操作。

# URLSearchParams

`createURLSearchParamsCodec` 把 Search Fields 变成可直接使用的 URLSearchParams codec。

```ts
import { createURLSearchParamsCodec } from '@decurl/react-router/codec';

const codec = createURLSearchParamsCodec(fields);
```

它提供两个方法：

```ts
codec.decode(searchParams);
codec.encode(values, options);
```

## Decode

```ts
const values = codec.decode(
  new URLSearchParams('?q=router&p=2&sort=latest'),
);
```

Decode 会遍历 Search Fields 中的每个字段：

- 使用字段的 canonical name 或 field property key 读取值。
- 如果配置了 alias，会按顺序尝试。
- single field 使用 `URLSearchParams.get(key)`。
- multi field 使用 `URLSearchParams.getAll(key)`。
- 缺失或解析失败时，回退到 `defaultValue` 或 `undefined`。

## Patch encode

`encode` 默认是 patch 语义。

```ts
const nextSearch = codec.encode(
  { page: 3 },
  { base: '?q=router&p=2&sort=latest' },
);

nextSearch.toString();
// q=router&sort=latest&page=3
```

Patch encode 的规则：

- 只处理 patch 中出现的 Search Fields field。
- patch 中不存在的 Search Fields field 不会被修改。
- patch 中不属于 Search Fields 的 key 会被忽略。
- `null` 或 `undefined` 表示删除对应 URL key。
- 写入 default value 时，默认删除对应 key。
- `preserveDefault: true` 可以保留 default value。

## Base search

`base` 用来表示当前 URLSearchParams。

```ts
codec.encode(
  { page: 2 },
  { base: new URLSearchParams('?q=router&debug=true') },
);
```

结果会保留 Search Fields 未触碰的参数：

```txt
q=router&debug=true&page=2
```

这对真实页面很重要，因为 URL 里可能有当前组件不负责的参数。

## 删除字段

```ts
codec.encode(
  { q: undefined },
  { base: '?q=router&page=2' },
).toString();
// page=2
```

`null` 和 `undefined` 都表示删除字段。对带 alias 的字段，删除会移除所有 alias key。

## 多值字段

```ts
const fields = defineFields({
  tag: field({
    mode: 'multi',
    decode: (values) => values.filter(Boolean),
  }),
});

const codec = createURLSearchParamsCodec(fields);

codec.encode({ tag: ['react', 'router'] }).toString();
// tag=react&tag=router
```

multi field 会按数组顺序写入多个同名 key。
