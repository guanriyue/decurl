# 内置默认模型

这个示例使用 `useSearchPagination()` 的默认分页模型。Page size 选择器直接消费 `useSearchPagination.pageSizeOptions`；修改 page size 时，页码默认回到第 1 页。

这个示例只在页面初始化时检查一次页码溢出。这里可以这样做，是因为数据总数固定，并且 Pagination UI 会在调用 `setPage` 前限制目标页码；修改 page size 时，默认策略也会把页码重置为 `1`。因此，组件挂载后不会通过当前交互产生新的越界页。

你可以尝试手动修改当前页面的路由参数，比如 `?page=999`，页码会自动修正。

这种“一次初始化检查”只适用于当前这类静态数据场景。如果 total 会随筛选、请求或数据修改发生变化，应在获得与当前查询匹配的可信 total 后重新调用 `preventOverflow`，不能只依赖初始化 effect。
